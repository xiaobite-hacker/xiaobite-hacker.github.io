Github搜索泄露信息

Github经常使用，有时候在无意中会将铭感数据上传到github中

"site.com" keyword language:???? NOT www.site NOT test.site

site.com

这里你可以添加你想搜索的 可以是”site.com” “sub.site.com” 或仅仅是”site”

keyword

添加想要搜索的关键字

我喜欢用这几个
password — passwd — pwd — secret — private — Ldap

因为如果我用像 secret 这样的关键词，我能得到有关的所有结果

https://github.com/techgaun/github-dorks/blob/master/github-dorks.txt

Language:

如果你搜索后东西有点多，可以通过language来筛选，比如
bash python ruby java
就像这样
"bugcrowd.com" password language:bash

NOT

如果你在搜索中有www.bugcrowd.com 大量出现，让你感到很烦，你可以使用NOT，就像这样
"bugcrowd.com" NOT www.bugcrowd NOT test.bugcrowd

"paypal" language:python password NOT sandbox.paypal NOT api.paypal NOT www.paypal NOT gmail.com NOT yahoo.com NOT hotmail.com NOT test

例子：
Org
org:bugcrowd password NOT www.bugcrowd
User
user:orwagodfather secret
github page
"bugcrowd.com" language:python secret NOT www.bugcrowd

我的技巧、方法与建议

1

目标范围：bugcrowd.com
超出范围：blog.bugcrowd.com

跳出框架思考
泄露的数据一直在范围内
所以开始搜索所有属于Bugcrowd的东西吧
像这样的，*.bugcrowd.*

2

始终关注最近的日期 Recently updated

3

在任何时候，如果你找到一些内容了，你可以在Linkdin上查看该用户是否是为该公司工作，有些员工在Github上会隐藏自己的工作地点信息，但是在Linkedin上不会
如果你真想知道他是否在Bugcrowd工作，或者在Google工作，那你可以发消息问问他

搜索用户的工作信息

user:orwagodfather linkedin
user:orwagodfather full name
user:orwagodfather https://
user:orwagodfather Ldap

如何想搜索公司内部的一些链接，可以这样

org:bugcrowd https://
org:bugcrowd host:

有时你会遇到一些内部链接，该链接指向的是一个特殊或重要的页面
它不受用户名和密码的保护，或者它需要默认凭据，比如 admin admin
或者 admin password，使用这些凭据，你将获得完全访问权限，并能轻松生成一份很棒的 P1 报告

4

有些公司特别是在那段时期依赖云服务进行远程办公，因此当你在 GitHub 和类似服务上搜索凭据时，请记住以下示例：
“centrify.com” password
service-now.com password
你也可以尝试搜索“send_keys”这个关键词。

5

github+翻译
如果你要搜索俄罗斯程序的漏洞
例如，你可以使用俄语的关键词
或者如果你要搜索来自意大利的用户
你可以使用意大利语的关键词
这样能得到很好的结果

host
user
pass

例子

“target.atlassian” password or secret or any keyword
“target.okta” password or secret or any keyword
“corp.target” password or secret or any keyword
“jira.target” password or secret or any keyword
“target.onelogin” password or secret or any keyword

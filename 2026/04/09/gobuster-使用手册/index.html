一、概述

Gobuster 是用 Go 编写的高性能目录/DNS/VHost 爆破工具，支持 7 种模式，专为渗透测试和安全评估设计。

核心特点：

  • ⚡ 多线程：可配置并发数
  • 🔍 7 种模式：dir / dns / vhost / fuzz / s3 / gcs / tftp
  • 📦 轻量：单二进制文件，无依赖
  • 🔧 模式扫描：支持模板化 URL 生成
  • 🐳 Docker 支持

三、基本语法与模式总览

模式 用途 示例
dir 目录/文件扫描 gobuster dir -u URL -w wordlist
dns DNS 子域名爆破 gobuster dns -d domain -w wordlist
vhost 虚拟主机发现 gobuster vhost -u URL -w wordlist
fuzz 自定义 Fuzzing gobuster fuzz -u URL/FUZZ -w wordlist
s3 AWS S3 存储桶枚举 gobuster s3 -w wordlist
gcs Google Cloud Storage 枚举 gobuster gcs -w wordlist
tftp TFTP 文件发现 gobuster tftp -s IP -w wordlist

十五、Gobuster vs ffuf vs Feroxbuster 对比

维度 Gobuster ffuf Feroxbuster
语言 Go Go Rust
模式数 7 种 1 种（FUZZ 万能） 1 种
目录扫描 ✅ ✅ ✅
DNS 爆破 ✅ ❌ ❌
VHost 发现 ✅ ✅ ❌
S3/GCS 枚举 ✅ ❌ ❌
TFTP 扫描 ✅ ❌ ❌
FUZZ 关键字 🟡 fuzz 模式 ✅ 任意位置 ❌
递归扫描 ❌ ✅ ✅（更强）
过滤器 🟡 基础 ✅ 6 种维度 ✅ 丰富
自动校准 ❌ ✅ ❌
模式扫描 ✅ ❌ ❌
速度 ⚡ 快 ⚡ 极快 ⚡ 极快
学习曲线 🟢 最低 🟡 中 🟢 低
适用场景 简单快速 + 多协议 全能 Fuzzer 递归目录扫描

选择建议：

  • 🚀 简单快速 + DNS/云存储：Gobuster（7 种模式最全面）
  • 🎯 灵活 Fuzzing：ffuf
  • 🔄 深度递归扫描：Feroxbuster

总结：Gobuster 的核心优势是多模式 + 简单易用。

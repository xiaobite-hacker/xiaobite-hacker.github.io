1. 框架架构与核心概念

1.1 六大模块类型

模块 目录 说明
Exploits modules/exploits/ 漏洞利用模块，攻击入口
Payloads modules/payloads/ 攻击载荷，Exploit 成功后执行的代码
Auxiliary modules/auxiliary/ 辅助模块：扫描、嗅探、枚举、fuzz
Post modules/post/ 后渗透模块：信息收集、提权、持久化
Encoders modules/encoders/ 编码器：混淆 Payload 绕过检测
Evasion modules/evasion/ 规避模块：绕过 AV/EDR
NOPs modules/nops/ 空指令：内存对齐填充

1.2 核心术语

术语 含义
Exploit 利用目标漏洞的代码
Payload Exploit 成功后执行的代码（Shell、Meterpreter 等）
Shellcode 机器码级别的 Payload
Module MSF 中可加载的功能单元
Session 与目标建立的交互会话
Listener/Handler 接收反向连接的监听服务
Stager 分阶段 Payload 的第一阶段（建立连接）
Stage 分阶段 Payload 的第二阶段（完整功能）
Single 单体 Payload（完整功能，不依赖网络传输 Stage）
LHOST / LPORT 攻击机 IP / 监听端口
RHOST / RPORT 目标 IP / 目标端口

6. Payload 体系详解

6.1 三种 Payload 类型

Singles（单体载荷）：

  • 独立完整，不依赖网络传输后续代码
  • 示例：windows/shell/reverse_tcp（完整的反弹 shell）
  • 命名特征：通常以 reverse_tcp / bind_tcp 结尾，没有分层

Stagers（初始阶段载荷）：

  • 建立连接的”引导程序”
  • 体积小，适合有限空间
  • 示例：windows/meterpreter/reverse_tcp（引导 Meterpreter）
  • 命名特征：在 stages 目录下的 payload

Stages（后续阶段载荷）：

  • 由 Stager 加载，提供完整功能
  • 示例：meterpreter（完整 Meterpreter 功能）
  • 体积较大，通过 Stager 传输

6.2 连接模式

模式 说明 适用场景
Reverse TCP 目标反连攻击机 最常用，穿透 NAT
Reverse HTTP 目标通过 HTTP 反连 穿透防火墙
Reverse HTTPS 目标通过 HTTPS 反连 加密通信，最隐蔽
Bind TCP 攻击机连接目标 目标有公网 IP
Bind Named Pipe 通过命名管道 SMB 通道

7.2 常用参数

参数 说明
-p PAYLOAD 指定 Payload

-f FORMAT 输出格式
-e ENCODER 指定编码器

-s SIZE 生成的最大 Payload 大小
--smallest 尽可能生成最小 Payload
--encrypt TYPE 加密 Payload
--encrypt-key KEY 加密密钥
-c FILE 指定自定义 shellcode 文件
-x FILE 指定模板文件（将 Payload 注入正常程序）
-k FILE 保留模板程序原有功能

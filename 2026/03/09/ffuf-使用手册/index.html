一、概述

ffuf (Fuzz Faster U Fool) 是用 Go 编写的高性能 Web Fuzzing 工具，通过 FUZZ 关键字可在 HTTP 请求的任意位置进行模糊测试。在渗透测试中，ffuf 用于发现隐藏目录、文件、参数、子域名等攻击面。

核心特点：

  • ⚡ 极快：高并发协程，远超 gobuster / wfuzz
  • 🎯 万能关键字：URL / Header / POST 数据任意位置放置 FUZZ
  • 🔍 6 维度过滤：状态码、大小、行数、单词数、正则、响应时间
  • 🤖 自动校准：-ac 自动检测并排除噪声响应（自定义 404 页面等）
  • 📦 递归扫描：自动发现子目录并继续深入
  • 🔧 外部编码器：支持 URL 编码、Base64 编码等
  • 🎛 多字典模式：clusterbomb / pitchfork 组合测试

十一、匹配器与过滤器详解

11.1 匹配器（保留结果）

参数 条件 示例
-mc HTTP 状态码 -mc 200,301,302 或 -mc all
-ms 响应大小（字节） -ms 1024
-mw 单词数 -mw 42
-ml 行数 -ml 15
-mr 正则表达式 -mr "admin|dashboard"
-mt 响应时间（ms） -mt ">5000"

-mc 默认值：200-299,301,302,307,401,403,405,500

参数 条件 示例

-fw 单词数 -fw 42
-fl 行数 -fl 15
-fr 正则表达式 -fr "Not Found"
-ft 响应时间 -ft ">3000"
-fmode 过滤逻辑 -fmode and

11.3 数值范围语法

11.4 执行逻辑

ffuf 内部处理顺序：

  1. 发送请求
  2. 接收响应
  3. 匹配器检查 → 不匹配则丢弃
  4. 过滤器检查 → 匹配则丢弃
  5. -replay-proxy 存在则转发匹配请求到代理
  6. 显示结果并保存

11.5 自动校准（-ac）

21.6 -H 参数追加而非覆盖

-H 在命令行和配置文件中是追加关系，不是覆盖。若配置文件有 -H "Cookie: x=1"，命令行再加 -H "Cookie: x=2" 会发送两个 Cookie 头。

21.7 RESTful 风格路径后缀问题

RESTful API 路径（如 /api/users/1）通常没有文件后缀，此时 -e 扩展名扫描无效。应直接用目录字典扫描路径段。

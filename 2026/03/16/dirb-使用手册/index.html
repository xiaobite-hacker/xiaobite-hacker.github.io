一、概述

dirb 是一款经典的 Web Content Scanner（Web 内容扫描器），通过基于字典的攻击方式扫描 Web 服务器上已存在或隐藏的 Web 对象（目录和文件），并分析响应结果。

核心特点：

  • 📦 Kali 内置：预装于 Kali Linux
  • 🔍 字典驱动：自带多套字典，也支持自定义
  • 📂 递归扫描：发现目录后自动深入扫描
  • 🛠 轻量简单：参数少，上手快
  • 📊 404 智能检测：自动识别自定义 404 页面
  • 🔗 配套工具：附带 dirb-gendict（字典生成器）和 html2dic（HTML 提词器）

定位：dirb 是内容扫描器而非漏洞扫描器——它帮你找到隐藏的目录和文件，但不检测其中的漏洞。

三、基本语法与参数总览

3.1 基本语法

  • <url_base>：目标 URL（必须以 / 结尾，除非使用 -t）
  • [<wordlist_file(s)>]：字典文件路径（可选，不指定则使用默认字典）
  • [options]：扫描选项

3.2 参数总览

参数 说明 默认值
-a <agent> 自定义 User-Agent Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)
-b 不压缩 URL 中的 /../ 和 /./ 序列 压缩
-c <cookie> 设置 Cookie 无
-E <cert> 使用客户端证书文件 无
-f 微调 404 检测（更精确地识别自定义 404） 关闭
-H <header> 添加自定义请求头 无
-i 不区分大小写搜索 区分大小写
-l 显示发现的 Location 重定向头 不显示
-N <code> 忽略指定 HTTP 状态码的响应 无
-o <file> 保存输出到文件 无
-p <proxy:port> 使用代理（默认端口 1080） 无
-P <user:pass> 代理认证 无
-r 不递归扫描 默认递归
-R 交互式递归（手动选择要扫描的目录） 关闭
-S 静默模式（不显示测试的词） 显示
-t 不强制 URL 末尾添加 / 强制添加
-u <user:pass> HTTP Basic 认证 无
-v 显示不存在的页面（404 也显示） 不显示
-w 遇到警告不停止 停止
-x <file> 从文件加载扩展名列表 无
-X <ext> 直接指定扩展名（如 .php,.html） 无
-z <ms> 请求间延迟（毫秒） 无延迟

十一、附带工具

dirb 附带两个实用工具：

11.1 dirb-gendict（字典生成器）

按模式生成字典，支持数字、字母、十六进制等。

11.2 html2dic（HTML 提词器）

从 HTML 页面中提取单词，生成字典。

选择建议：

  • 🚀 快速入门 + 简单扫描：dirb（参数少，自带字典）
  • 🎯 灵活 Fuzzing + 复杂过滤：ffuf
  • ⚡ 多协议 + DNS 爆破：gobuster
  • 🔄 深度递归 + 交互控制：dirb -R 或 ffuf 交互模式

典型工作流：用 dirb 做快速初筛 → 发现可疑目录 → 用 ffuf 深入测试

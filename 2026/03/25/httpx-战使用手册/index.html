一、概述

httpx 是 ProjectDiscovery 开发的快速多功能 HTTP 工具包，用于对大量目标进行 HTTP 探测，收集存活主机的状态码、标题、技术指纹、IP、CDN、TLS 等信息。

核心特点：

  • ⚡ 极速探测：高并发 + retryablehttp 库，兼顾速度与稳定性
  • 🎯 智能回退：自动从 HTTPS 回退到 HTTP
  • 🔍 全面探针：状态码 / 标题 / 技术栈 / Favicon / TLS / CDN / ASN / IP
  • 🖥 网页截图：无头浏览器截图
  • 📊 丰富输出：JSONL / CSV / 数据库（MongoDB / PostgreSQL / MySQL）
  • 🔗 生态集成：与 subfinder / nuclei / katana 等 ProjectDiscovery 工具无缝联动
  • 🛡 WAF 友好：重试、退避、限速等反封禁机制

注意：Kali 中包名为 httpx-toolkit（避免与 Python 的 httpx 库冲突）。

输入

参数 说明
-u, -target 目标主机（可多次）

-rr, -request 原始 HTTP 请求文件

信息探针（Probes）

参数 说明
-sc, -status-code 显示状态码
-cl, -content-length 显示响应大小
-ct, -content-type 显示 Content-Type
-title 显示页面标题
-server, -web-server 显示 Server 头
-td, -tech-detect 技术栈检测（Wappalyzer）
-rt, -response-time 显示响应时间
-lc, -line-count 显示响应行数
-wc, -word-count 显示响应单词数
-location 显示重定向 Location
-method 显示请求方法
-ip 显示主机 IP
-cname 显示 CNAME
-cdn 检测 CDN/WAF
-favicon Favicon mmh3 hash
-jarm JARM 指纹
-hash 响应体 hash（md5/mmh3/simhash/sha1/sha256/sha512）
-bp, -body-preview 显示响应体前 N 字符（默认 100）
-ws, -websocket 检测 WebSocket
-probe 显示探测状态
-asn 显示 ASN 信息
-extract-fqdn, -efqdn 从响应中提取域名和子域名
-pipeline 检测 HTTP/1.1 Pipeline
-http2 检测 HTTP/2
-vhost VHost 探测

匹配器

参数 说明
-mc 匹配状态码
-ml 匹配响应大小
-mlc 匹配行数
-mwc 匹配单词数
-mfc 匹配 Favicon hash
-ms 匹配字符串
-mr 匹配正则
-mcdn 匹配 CDN 提供商
-mrt 匹配响应时间
-mdc DSL 表达式匹配

过滤器

参数 说明
-fc 过滤状态码
-fl 过滤响应大小
-flc 过滤行数
-fwc 过滤单词数
-ffc 过滤 Favicon hash
-fs 过滤字符串
-fe 过滤正则
-fcdn 过滤 CDN 提供商
-frt 过滤响应时间
-fpt 过滤页面类型（login/captcha/parked）
-fd 过滤重复响应

提取器

参数 说明
-er 正则提取响应内容
-ep 预设正则提取（url/ipv4/mail）

速率限制

参数 说明 默认值
-t, -threads 线程数 50
-rl, -rate-limit 每秒最大请求数 150

请求配置

参数 说明
-H, -header 自定义 Header（可多次）
-x 请求方法（all 探测所有方法）
-body POST 请求体
-fr, -follow-redirects 跟随重定向
-maxr, -max-redirects 最大重定向数（默认 10）
-fhr, -follow-host-redirects 仅跟随同主机重定向
-random-agent 随机 UA（默认 true）
-auto-referer 自动设置 Referer

连接配置

参数 说明
-http-proxy, -proxy 代理（HTTP/SOCKS）
-sni 自定义 TLS SNI
-r, -resolvers 自定义 DNS 解析器
-allow 允许的 IP/CIDR
-deny 拒绝的 IP/CIDR
-timeout 超时秒数（默认 10）
-retries 重试次数
-delay 请求间延迟
-resume 恢复扫描

杂项

参数 说明
-p, -ports 端口（nmap 语法，如 80,443,8000-9000）
-path 路径或路径列表
-tls-grab 抓取 TLS 证书信息
-tls-probe 对 TLS 证书中的域名发送探测
-csp-probe 对 CSP 头中的域名发送探测
-pa, -probe-all-ips 探测所有关联 IP
-nf, -no-fallback 不自动回退，同时显示 HTTP 和 HTTPS
-nfs, -no-fallback-scheme 用输入中的协议探测
-s, -stream 流式模式（不排序，边读边处理）
-exclude 排除（cdn / private-ips / CIDR / IP / 正则）
-ldp, -leave-default-ports 保留默认端口号
-ztls 使用 ztls 库
-tlsi, -tls-impersonate TLS JA3 随机化

输出

参数 说明
-o, -output 输出文件
-oa, -output-all 所有格式输出
-j, -json JSONL 格式
-csv CSV 格式
-sr, -store-response 保存 HTTP 响应
-srd 响应保存目录
-ob, -omit-body 省略响应体
-irh JSON 中包含响应头
-irr JSON 中包含完整请求/响应
-irrb JSON 中包含 base64 请求/响应
-include-chain JSON 中包含重定向链
-rdb 存入数据库
-rdbt 数据库类型（mongodb/postgres/mysql）

截图（无头浏览器）

参数 说明
-ss, -screenshot 启用截图
-system-chrome 使用本地 Chrome
-esb JSON 中排除截图字节
-no-screenshot-full-page 不截全页
-st 截图超时（默认 10s）
-sid 截图前空闲时间（默认 1s）
-jsc 导航后执行 JavaScript

调试

参数 说明

-debug-req 显示发送的请求

十八、httpx vs 其他工具对比

维度 httpx 目录扫描工具（ffuf/dirb/dirsearch）
核心功能 HTTP 探测 / 信息收集 目录/文件发现
输入 主机/IP/CIDR 列表 URL + 字典
输出 状态码/标题/技术栈/IP/CDN/TLS 目录/文件路径
速度 ⚡ 极快 ⚡ 快
截图 ✅ 无头浏览器 ❌
技术检测 ✅ Wappalyzer 指纹 ❌
TLS 分析 ✅ JARM/证书/CSP ❌
管道联动 ✅ subfinder/nuclei/katana ✅ 互相联动
数据库输出 ✅ MongoDB/PostgreSQL/MySQL ❌
适用阶段 侦察 / 存活验证 目录枚举

典型工作流：subfinder → httpx（存活验证 + 信息收集）→ ffuf/dirsearch（目录扫描）→ nuclei（漏洞检测）

1. DNS 隧道原理

1.1 基本思想

将非 DNS 数据编码后封装在 DNS 查询/响应中，利用 DNS 协议的普遍通行性（几乎所有网络都允许 DNS 出站）建立隐蔽通信通道。

1.2 数据编码方式

编码方式 效率 使用工具
Base32 60% iodine (默认)
Base64 75% dnscat2
Base128 87.5% iodine
Hex 50% 部分工具
自定义编码 变化 Heyoka 等

1.3 DNS 记录类型利用

记录类型 数据容量 传输方向 常用工具
TXT 最大 255 字节/条 下行 iodine, dnscat2
NULL 255 字节 (私有类型) 上/下行 iodine
CNAME 255 字节 下行 iodine
A 4 字节 (IPv4) 下行 iodine
AAAA 16 字节 (IPv6) 下行 iodine
MX 255 字节 下行 iodine
SRV 255 字节 下行 iodine
TXT 255 字节 上/下行 dnscat2

6. DNS-Shell 与 ICMP 隧道对比

6.1 DNS-Shell

  • Python 编写的 DNS 命令执行工具
  • 通过 TXT 记录传输命令和回显
  • 无需虚拟网卡，直接执行命令
  • 适合单向控制场景

6.2 与 ICMP 隧道对比

维度 DNS 隧道 ICMP 隧道
协议 DNS (UDP/53) ICMP (ping)
通行性 ★★★★★ 几乎所有网络 ★★★★☆ 部分网络拦截
带宽 较低 (255 字节/query) 极低 (受限于 ping 包)
隐蔽性 ★★★★☆ ★★★☆☆

11. 规避与对抗

11.1 攻击侧规避手法

手法 原理 效果
降低查询频率 增大 sleep 时间 绕过频率检测
混合正常查询 隧道查询中夹杂正常 DNS 查询 降低 TXT 占比
使用 A 记录 改用 A 记录（每次 4 字节） 绕过 TXT 异常检测
分段传输 每次只传输少量数据 降低子域名长度
自定义编码 替换 Base32/64 为自定义映射 绕过编码特征匹配
域名伪装 使用类似 CDN/云服务的域名 混淆目标域名
DoH/DoT DNS over HTTPS/TLS 加密 DNS 流量
多域名轮询 使用多个子域名分散 增加检测难度
子域名压缩 优化编码效率减少子域名长度 绕过长度检测
添加噪声 发送大量无意义的正常查询 淹没真实隧道流量

11.2 蓝队对抗策略

11.3 防御部署建议

  1. DNS 日志全量采集：部署 dnstap / DNS query logging
  2. 基线建模：统计各主机正常 DNS 行为（频率、类型分布、域名分布）
  3. 实时告警：熵值 + 频率 + 类型占比 多维度联合告警
  4. DNS 策略限制：阻断到非授权 DNS 服务器的查询（强制内部 DNS 代理）
  5. 被动 DNS 监控：关联外部 Passive DNS 数据
  6. 终端检测：检测 TUN/TAP 网卡创建行为
  7. 流量基线：建立 DNS 占总流量比例的基线

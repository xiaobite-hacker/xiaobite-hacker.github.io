级别 输出内容

2. 目标指定参数

参数 说明 示例
-u URL 目标 URL sqlmap -u "http://t.com/?id=1"
-d DIRECT 直连数据库 sqlmap -d "mysql://root:pass@host:3306/db"
-r REQUESTFILE 从文件读取 HTTP 请求 sqlmap -r request.txt
-m BULKFILE 批量目标 URL 列表 sqlmap -m urls.txt
-g GOOGLEDORK Google 搜索结果作为目标 sqlmap -g "inurl:\".php?id=\""
-c CONFIGFILE 从配置文件读取选项 sqlmap -c sqlmap.conf
--url=URL 同 -u

-d 直连数据库格式：

  • MySQL/Oracle/MSSQL/PostgreSQL：DBMS://USER:PASSWORD@HOST:PORT/DBNAME
  • SQLite/Access/Firebird：DBMS://DBFILE_PATH

3. 请求控制参数

3.1 基本请求控制

参数 说明 示例
-A AGENT 自定义 User-Agent -A "Mozilla/5.0 ..."
--random-agent 随机 User-Agent（强烈建议开启）
-H HEADER 添加额外 HTTP 头 -H "X-Forwarded-For: 127.0.0.1"
--method=METHOD 强制 HTTP 方法 --method=PUT
--data=DATA POST 数据 --data="id=1&name=test"
--param-del 参数分隔符（默认 &） --param-del=";"
--cookie=COOKIE 指定 Cookie --cookie="PHPSESSID=abc123"
--cookie-del Cookie 分隔符 --cookie-del=";"
--referer=REFERER 指定 Referer --referer="http://google.com"
--proxy=PROXY 使用代理 --proxy="http://127.0.0.1:8080"
--force-ssl 强制 HTTPS
--chunked HTTP 分块传输编码
--hpp HTTP 参数污染
--eval=EVALCODE 请求前执行 Python 代码 --eval="import hashlib; h=hashlib.md5(id.encode()).hexdigest()"

3.4 Cookie 自动处理

SQLMap 遇到 Set-Cookie 响应头会自动在后续请求中携带，且会自动测试 Cookie 值是否存在注入。

关闭自动携带：--drop-set-cookie

4. 检测与注入配置

4.1 Level & Risk

--level（1-5，默认 1）：

Level 测试范围
1 GET/POST 参数
2 + Cookie
3 + User-Agent、Referer
4-5 更多边界和 Payload

--risk（1-3，默认 1）：

Risk 说明
1 安全 Payload
2 + 时间盲注 Payload
3 + OR 布尔盲注 Payload（可能修改数据！）

⚠️ 注意：在 UPDATE 等写操作的注入点，--risk=3 可能导致全表数据更新，务必谨慎！

12. 常见问题

Q: SSL 连接失败

解决：添加 --force-ssl 并挂代理（如 Burp）

Q: 被 302 重定向到登录页

解决：使用 -r 抓包文件并包含 Cookie，或手动指定 --cookie

Q: 注入速度太慢

解决：

  • 增加线程 --threads=10
  • 开启 --null-connection
  • 使用 --predict-output
  • 缩小测试范围 -p 指定参数

Q: SQLMap 默认 UA 太明显被识别

解决：--random-agent 必开

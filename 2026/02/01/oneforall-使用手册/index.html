一、概述

OneForAll 是一款国产、功能全面的子域收集工具，集百家之长，融合了 60+ 收集模块、子域爆破、子域验证、子域爬取、子域置换、子域接管检查等功能于一体。

核心特点：

  • 📦 60+ 收集模块：证书透明度、DNS 数据集、搜索引擎、威胁情报、爬虫档案、DNS 查询
  • 💣 子域爆破：基于 massdns，每秒 35 万+ DNS 解析，支持递归爆破、fuzz 模式
  • ✅ 子域验证：DNS 解析 + HTTP 请求 + title/banner 获取
  • 🔍 子域爬取：从响应体和 JS 中二次发现子域
  • 🔄 子域置换：利用 altdns 技术重组子域名发现新子域
  • ⚠️ 子域接管检查：检测子域接管风险（支持自动接管）
  • 🌐 CDN 检测：识别 CDN 节点
  • 🏷️ Web 指纹识别：识别 Web 技术栈
  • 🗄️ SQLite3 数据库：持久化存储，支持增量对比

四、命令行参数详解

核心参数

参数 说明 默认值 示例
--target 单个目标域名（必填） — --target example.com
--targets 批量域名文件（必填） — --targets domains.txt
--brute 是否开启子域爆破 False --brute True
--dns 是否进行 DNS 解析 True --dns True
--req 是否进行 HTTP 请求 True --req True
--port HTTP 请求端口范围 small --port small / medium / large
--valid 只导出存活子域 False --valid True
--fmt 结果保存格式 csv --fmt csv / json / txt
--path 结果保存路径 None（自动） --path /tmp/results
--takeover 是否检查子域接管 False --takeover True
--show 是否在终端显示结果 False --show True

端口范围说明

参数值 说明
small 仅探测 80、443 端口（默认）
medium 探测常见 Web 端口
large 探测大量端口

最佳实践：三者联合

总结：OneForAll 是”瑞士军刀”级别的子域收集工具，适合需要一站式全面侦察的场景。如果你的流程是自动化的（流水线），Subfinder 更合适。如果需要深度且有持久化需求，Amass 更合适。三者联合使用，取并集，是最完整的方案。

2. 蚁剑（AntSword）

2.1 工具简介

蚁剑是开源的跨平台 WebShell 管理工具，基于菜刀改进而来，支持编码器/解码器对流量进行混淆。

2.2 WebShell 文件

语言 特征
PHP 使用 assert、eval 执行
ASP 使用 eval 执行
JSP 使用 Java 类加载（ClassLoader），含 base64 编解码

2.3 流量特征

蚁剑内置了多个UA头，每次执行命令都会更换UA头；

3. 冰蝎（Behinder）

3.1 工具简介

冰蝎是一款基于 AES 加密的动态二进制 WebShell 管理工具，通信全程加密，是目前检测难度最高的工具之一。

3.2 WebShell 文件

语言 特征
PHP 判断是否开启 openssl 选择加密算法；含 eval/assert；16位密钥变量（默认 k）
ASP for 循环中进行异或处理
JSP 利用 Java 反射，含 ClassLoader、getClass().getClassLoader()

共同特征：采用预共享密钥，密钥格式为 md5(连接密码)[0:16]，默认密码 rebeyond → 密钥 e45e329feb5d925b。

3.3 流量特征

冰蝎 2.0

加密方式 AES16 + Base64
User-Agent 内置17个 UA，每次随机选择，同一 IP 的 UA 频繁变换是强特征

Cookie Cookie: PHPSESSID=; path=/（空 PHPSESSID）

| Accept text/html, image/gif, image/jpeg, *; q=.2, */*; q=.2

默认情况请求头和响应头都会有Connection: Keep-Alive

冰蝎 3.0

密钥 AES加密的密钥为webshell连接密码的MD5的前16位，默认连接密码是”rebeyond”(即密钥是md5(‘rebeyond’)[0:16]=e45e329feb5d925b)

Content-Length 命令执行功能时，Content-Length 固定为 5740 或 5720
缓存控制 请求头含 Pragma: no-cache、Cache-Control: no-cache
Accept text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9
Content-Type 可以反编译修改，默认Content-Type: application/octet-stream

User-Agent

冰蝎 4.0

Accept（弱） application/json, text/javascript, */*; q=0.01

缓存控制 请求头含 Pragma: no-cache、Cache-Control: no-cache

Content-Type（弱） Application/x-www-form-urlencoded、Application/octet-stream
UA 仍内置多 UA 随机切换

长连接 默认使用 Connection: Keep-Alive
冰蝎通讯默认使用长连接，避免了频繁的握手造成的资源开销。默认情况下，请求头和响应头里会带有 Connection。

本地端口 Java 连接使用本地端口从 ~49700 开始，每次递增

流量特征固定的请求头和响应头

1.PHP站点默认口令Default_xor_base64协议加密流量特征，请求字节头：dFAXQV1LORcHRQtLRlwMAhwFTAg/M

响应字节头：TxcWR1NNExZAD0ZaAWMIPAZjH1BFBFtHThcJSlUXWEd

2.PHP站点默认口令Default_aes协议加密流量特征，请求字节头：m7nCS8n4OZG9akdDlxm6OdJevs/jYQ5/IcXK

响应字节头：mAUYLzmqn5QPDkyI5lvSp6DmrC24FW39Y4YsJhUqS7

3.JSP站点默认口令Default_xor_base64协议，aes_with_magic协议，Default_aes协议,加密流量特征，响应字节头：QhoVQgMXEUcUCBMHAGFZaQtuHFUVXlkWGhBcF1QVCRJ

默认密钥 仍可能出现 e45e329feb5d925b
默认时，所有冰蝎4.0 webshell都有“e45e329feb5d925b” 一串密钥。该密钥为连接密码32位md5值的前16位，默认连接密码rebeyond

3.4 检测要点

  • 同一 IP 短时间内 UA 频繁变化
  • Content-Length 固定值（5740/5720）
  • 请求或响应中出现固定字节头签名
  • 空 Cookie 或 Cookie 异常
  • 频繁出现 Pragma: no-cache + Cache-Control: no-cache 组合
  • 本地端口从 49700 开始依次递增

4. 哥斯拉（Godzilla）

4.1 工具简介

哥斯拉是继冰蝎之后的新一代 WebShell 管具，支持多种加密方式和自定义 HTTP 头部，采用了类似冰蝎 2.0 的密钥交换机制。

4.2 WebShell 文件

语言 特征
JSP（默认编码） 出现 xc、pass 字符；Java 反射（ClassLoader、getClass().getClassLoader()）；Base64 编解码
PHP/ASP（默认） 为普通一句话木马形式

4.3 流量特征

牔征项 描述 强度
Accept（请求） text/html,image/gif,image/jpeg,*; q=.2, */*; q=.2 弱（可自定义）
Cache-Control（响应） no-store, no-cache, must-revalidate 弱（可自定义）
Cookie 末尾分号 所有请求的 Cookie 值末尾都带有一个多余的分号 ; 强特征
响应体结构 32 位 MD5 字符串按前 16 位 + Base64 数据 + 后 16 位 拆分拼接 强特征
